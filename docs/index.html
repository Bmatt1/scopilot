API Routes

GET /health

Health check. Returns { "status": "healthy" }. Used by Render deploy health checks.

POST /api/scope/submit

Create a new lead from a homeowner project scope submission.

All fields optional except address, homeowner_name, homeowner_email.

Request body

{
  "address": "123 Main St, Owensboro KY 42301",
  "latitude": 37.7719,
  "longitude": -87.1117,
  "project_area_geojson": { "type": "Polygon", "coordinates": [...] },
  "sq_footage": 850,
  "project_type": "concrete",
  "trade_inputs": { "tear_out": "yes", "thickness": "4in" },
  "notes": "Cracks in existing driveway",
  "homeowner_name": "Jane Doe",
  "homeowner_email": "jane@example.com",
  "homeowner_phone": "270-555-0142",
  "photo_urls": ["https://pub-...r2.dev/photo1.jpg"]
}

Legacy clients (pre-vertical-Q&A) can pass tear_out, reinforcement, finish_type, has_drainage as top-level fields. The server merges them into trade_inputs automatically.

Response 200

{ "success": true, "lead_id": 42, "estimate_low": 5100, "estimate_high": 8500 }
POST /api/scope/upload

Upload a photo. Returns a public R2 URL.

{ "data_url": "data:image/jpeg;base64,/9j/4AAQSkZJR...", "filename": "driveway.jpg" }

Without POLSIA_R2_BASE_URL set, returns the data URL directly — dev fallback only.

{ "url": "https://pub-...r2.dev/lead-photo-1716400000000.jpg" }

Short-lived passwordless login tokens. 15-minute expiry, rate-limited to 3 per hour per contractor.

Active login sessions. Managed by connect-pg-simple — you generally don't write to it directly. Each row holds the contractor ID a cookie maps to, plus an expiry.

Adding a new vertical: No migration needed. trade_inputs is JSONB — add the vertical to public/js/questions-config.js and it works immediately.

All schema changes go in migrations/ as timestamped JS files. Never in runtime code.

User Flows

Homeowner Flow

/scope.html

  1. Pick project type (8 verticals)
  2. Set location — address autocomplete or GPS pin drop. Result: lat/lng + address.
  3. Draw project area polygon on satellite map. Turf.js calculates sq_footage in real-time.
  4. Answer vertical-specific questions (from questions-config.js). Stored in trade_inputs.
  5. Upload up to 3 photos (via /api/scope/upload, proxied to R2).
  6. Enter contact info. Client-side ballpark estimate shown as a range.
  7. Submit — POST /api/scope/submit. Server recalculates estimate (never trusts client), saves lead, fires email notification (fire-and-forget), returns confirmation.

Why GPS-first + polygon: Rural properties often don't have a clean street address. Drawing the area is faster and more accurate than asking the homeowner to measure. The polygon is overlaid on the satellite view in the contractor email so the contractor can see the exact site.

Contractor Flow

/login → /contractor

  1. Contractor signs up at /signup or logs in at /login — by password, by one-click magic link, or after a password reset.
  2. Dashboard shows only that contractor's leads (filtered by status). Sidebar shows their scoping link, their claimed ZIPs, and a performance card.
  3. Click a lead card for full detail: satellite map, contact info, project specs (from trade_inputs + questions-config.js), photos, notes.
  4. Update status via PATCH /api/contractors/me/leads/:id/status, or "pass" a lead to the opportunity board via POST /api/leads/:id/pass.
  5. New leads arrive by email with full details + satellite map + polygon overlay + photos.

Auth model: Cookie-based sessions stored in Postgres (connect-pg-simple). Cookie is HttpOnly, SameSite=Lax, Secure in production. SESSION_SECRET env var required. The session ID is regenerated on every login event to block session-fixation attacks. There is no localStorage fallback — that workaround was removed; if cookies stop working, fix the cookie config, not the symptom.

Auth API (high level)

EndpointWhat it does
POST /api/auth/signupCreate a contractor account, log in.
POST /api/auth/loginEmail + password. Redirects to /contractor.
POST /api/auth/logoutEnd the session.
GET /api/auth/meReturns the logged-in contractor's profile.
POST /api/auth/magic-linkEmail a one-click login link. 15 minutes, 3 requests per hour per email.
GET /auth/magic?token=…The link clicked from the email. Logs the contractor in and redirects.
POST /api/auth/forgot-passwordEmail a 1-hour password reset link.
POST /api/auth/reset-passwordSet a new password and log in.
